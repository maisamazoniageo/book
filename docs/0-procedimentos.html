1  Procedimentos

1.1 Procedimentos Gerais

As camadas originais: glebas publicas, florestas publicas, unidades de conservação, terras indígenas, projetos de assentamentos, territórios quilombolas, certificação SIGEF e certificação SNCI devem ser analisadas no QGIS passando pelos seguintes procedimentos:

  • Eliminação de polígonos multipartes, essa geometria uni dois ou mais feições a uma mesma linha de atributos na tabela, esse formato costuma gerar muitos erros quando processados com cortes, sobreposições e uniões com outras camadas vetoriais. Algoritimo: Multipartes para partes simples;
  • Eliminação de feições duplicadas, ou seja, feições com geometrias exatamente iguais. Algoritimo: Verificador de Geometrias-> camada não deve ter duplicados, camada não deve ter geometrias multipartes e camada não deve ter geometrias inválidas;
  • Correção de geometria através da ferramenta Corrigir geometrias - Esse algoritmo tenta criar uma representação válida de uma dada geometria inválida sem perder nenhum dos vértices de entrada. Geometrias já válidas são retornadas sem intervenção adicional. Sempre gera uma camada de várias geometrias.;

1.2 Scripts

1.2.1 clip-temas-por-uf-e-dissolv.ipynb

Primeiro script a ser utilizado após a validação das camadas primárias. Ele irá recortar as camadas por unidade da federação gerando um arquivo com polígonos simples e outro com os polígonos dissolvidos.

Os arquivos de polígono simples serão utilizados para análises individuais nas quais cada polígono de um tema é estudado de forma individual ou a sua interação com os demais temas.

Os arquivos de polígonos dissolvidos serão utilizados para os cálculos de totalização, ou seja, total da camada na área de referência, seja ela o estado, município ou outra. Essa operação visa eliminar da totalização as sobreposições que ocerrem, principalmente em polígonos com feição de baixa precisão que não foram normalizados por uma certificação conforme a lei.

1.3 Fonte dos dados brutos

1.3.1 Imóveis Certificados

1.3.1.1 Extração

  • Ddos obtidos através da tabela tab_sigef_geral no banco de dados cartos de propriedade do INCRA;
  • Contém informações sobre imóveis certificados no SIGEF.
  • Data da extração: 28 de maio de 2028

1.3.1.2 Transformação

  • Os dados foram selecionados com base na poligonal da área de estudo sendo os demais apagados;
  • A camada foi convertida de multipoligon para poligon, ou seja, mutitiparte para parte simples;
  • Os polígonos que extrapolaram a poligonal da área de estudo foram recortados, conservando apenas a parte interna à área de estudo;
  • Criar um campo com a área calculada em hectares para registrar a área nova para os polígonos pelo fato do corte pela área de estudo. A coluna recebe a denominação de area_calc_ha.

1.3.1.3 Carregamento

  • Carregado na tabela sigef no banco de dados glebas-federais.gpk

1.3.2 Glebas Federais

1.3.2.1 Extração

  • Dados obttidos através da tabela tab_glebas_federais no banco de dados cartos de propriedade do INCRA;
  • Contém informações sobre as glebas federais;
  • Data de extração: 18 de maio de 2025

1.3.2.2 Transformação

  • Os dados foram selecionados com base na poligonal da área de estudo sendo os demais apagados;
  • A camada foi convertida de multipoligon para poligon, ou seja, mutitiparte para parte simples;
  • Os polígonos que extrapolaram a poligonal da área de estudo foram recortados, conservando apenas a parte interna à área de estudo;
  • Criar um campo com a área calculada em hectares para registrar a área nova para os polígonos pelo fato do corte pela área de estudo. A coluna recebe a denominação de area_calc_ha.

1.3.2.3 Carregamento

  • Carregado na tabela glebas_mais_amazonia no banco de dados glebas-federais.gpk

1.3.3 SNCI

1.3.3.1 Extração

  • carregado da tabela tb_snci_vigente do banco de dados cartos

1.3.3.2 Transformação

  • Acrescentado uma coluna de natureza onde foram utilizado dua categorias {‘Público’: ‘imóveis aparentemente ligados a entes públicos’, ‘Particular’: ‘imóveis não selecionados como públicos’};
  • foi utilizado uma consulta SQL com os parâmetros:
select * from snci where  
   "nome_interessado"  like 'INCRA%'
or "nome_interessado" like '%SUPERINTENDÊNCIA%'
OR "nome_interessado" like '%ESTADO%'
OR "nome_interessado" like '%ESTADO%'
OR "nome_interessado" like '%SR(15)%'
OR "nome_interessado" like '%SRFA%'
or "nome_interessado" like '%SUPERINTENDENCIA%'
or "nome_interessado" like '%UNIÃO'
or "nome_interessado" like '%UNIAO%'
or "nome_interessado" like '%UNIAO - INCRA%'
or "nome_interessado" like '%UNIAO FEDERAL%'
or "nome_interessado" like '%UNIÃO FEDERAL%'
or "nome_interessado" like '%SECRETARIA%'
